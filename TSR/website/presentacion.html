Objectivos Asignatura

  • Objectivo General
    • Aproximación pragmática al diseño y construcción de Sistemas Distribuidos (SD)
  • Objectivos específicos
    • Entender las propietades de los SD
      • Advantatges i àmbit d'aplicació, Inconvenients i aspectes a resoldre
      • Influència de l'estructura del sistema (arquitectura)
    • Coneixer les principals tecnologíes i aproximacions
      • Programació asincrònica per a la implantació de components
      • Middleware per a facilitar la cooperació entre components
    • Capacitar per al disseny de l'arquitectura adecuada per a problemes especifics
      • Estudi de examples de sistemes i la seua arquitectura
      • Ús de tecnologies rellevants per a la resolució de problemes de laboratori

Estructura Asignatura

  • Asignatura de 6 créditos
    • Teoría (1.5 cr)
      • Principios i propiedades generales
    • Seminarios (3 cr)
      • Tecnologías básicas, ejemplos, resolución de problemas
    • Laboratorio (1.5 cr)
      • Implantación de soluciones a problemas sencillos

Teoría y seminarios se imparten de forma conjunta: en el resto del texto se utilizas para ambos el nombre "Teoria"

Laboratorio

  • Proyecto 0 (JavaScript básico: desde la segunda semana). Sin supervisión
    • javaScript es distinto a Java
      • Diferente model de tipus, soporte en ejecución, etc.
      • Programación funcional
      • Programación dirigida por eventos (servidores asincrónicos)
  • 10 sesiones supervisadas, repartidas en tres proyectos
    1. Proxy inverso TCP/IP (3 sesiones). Utilita javaScript (JS) y NodeJS (Node)
    2. Aplicaciones en NodeJS y 0MQ (4 sesions) Utiliza JS, Node, 0MQ
    3. Despliegue (3 sesions). Utiliza JS, Node, Docker
