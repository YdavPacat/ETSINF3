  1. Presentación

    • Objectivos de la assignatura, y su contenido teórico y práctico
    • Detalla la organización docente y el calendario de trabajo
    • Establece la normativa de evaluación
    • Proporciona referencias al material de consulta

  2. Introducción a los Sistemas Distribuidos

    • Repasa el concepto de Sistema Distribuido (SD) (estudiado en la asignatura CSD)
    • Describe algunos ejemplos y áreas de aplicación
    • Repasa la evolución històrica, hasta Cloud COmputing (CC)
    • Compara las dos posibles alternativas para implantar servidores escalables:
      • Servidores concurrentes
      • Servidores asincrónicos
  3. NodeJS
    • Introduce el uso de JavaScript y NodeJS para implantar servidores asincrónicos
      • únicamente estudiamos la parte de JS necesaria para esta assignatura
    • Proporciona los contenidos previos necesarios para el desarrollo de les prácticas de laboratorio
  4. 0MQ
    • Repasa el concepto de middleware, especialmente el middleware orientado a comunicaciones
    • Introduce 0MQ como un ejemplo de middleware orientado a comunicaciones
      • Posteriormente se utilitza en las prácticas de laboratorio
      • La combinación de NodeJS y 0MQ permite construir servidores escalables y tolerantes a fallos con relativamente poco código
  5. Despliegue
    • La configuración y arranque de un SD requiere el despliegue de diversos components software en nodos distintos. Cada nodo debe coneixer algunos aspectos básicos del sistema para poder interactuar con otros componentes
    • Esta unidad temática presenta mecanismos para automatizar esas tareas (configurar la cantidad de instancias de cada componente, su ubicación, y la forma en que los distintos componentes interactuan entre sí
    • Introduce la herramienta Docker, que posteriormente utilizamos en el laboratorio
  6. Gestión de fallos
    • Caracteriza las situaciones de fallo en un SD
    • Estudia los modelos de replicación
    • Identifica los modelos de consistencia
  7. Escalabilidad
    • Identifica
      • Los mecanismos básicos para mejorar la escalabilidad
      • Los aspectos que pueden comprometer la escalabilidad (puntos de contención)
    • Trata la escalabilidad a nivel de datos (almacenes escalables $\rightarrow$ noSQL)
    • Presenta ejemplos de escalabilidad a distintos niveles
