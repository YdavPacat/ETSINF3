JSON: JavaScript Object Notation.

  • Para paso de mensajes o almacenamiento podemos usar texto

  • JSON = sintáxis estandard para almacenar/intercambiar datos en formato texto

  • Podemos usar JSON desde cualquier lenguaje de programación, pero aquí asumimos JavaScript)

    • JSON.stringify convierte un objecto JavaScript a JSON
    • JSON.parse reconstruye el objeto desde el texto de JSON

Sintaxis JSON

  • Deriva de la sintaxis para representar valores en JavaScript:
    • string: entre comillas dobles "name"
    • número: 3.26
    • boolean: true
    • null: null
    • array: secuencia de valores entre [] separados por comas
      • Admite cualquier valor 
        [1,2,"Pepe",[true,false]]
    • objeto: pares clave:valor entre {} y separados por comas
      • La clave(nombre o propiedad) es una tira
      • El valor es cualquier valor válido JSON 
        {"id":"14737B", "age":32, "addr":{"street":"St Michael", "num":14}}
  • A diferencia de js, JSON:
    • no soporta valores función, fecha, undefined
    • no admite comillas simples para definir tiras
    • no admite números o identificadores como claves en los objetos

Ejemplo

  // sending (JSON.stringify(obj)-> texto)
const fs = require('fs')
var myObj = {"name":"John", "age":31, "city":"New York"}
console.log(myObj)        // { name: 'John', age: 31, city: 'New York' }
var txt = JSON.stringify(myObj)
console.log(txt)          // {"name":"John","age":31,"city":"New York"}

  // receiving (JSON.parse(texto)->obj)
var obj = JSON.parse(txt)
console.log(obj)          // { name: 'John', age: 31, city: 'New York' } (como myObj)

  // Storing Data
fs.writeFileSync("testJSON", txt) // store obj
var txtFich = fs.readFileSync("testJSON") // retrieving obj
obj = JSON.parse(txtFich);
console.log(obj)          // { name: 'John', age: 31, city: 'New York' } (como myObj)

Comparación con XML

  • JSON y XML permiten intercambiar o almacenar valores en formato texto
    • Ambos son:
      • Auto-descriptivos (human readable), jerárquicos (values within values), y multilenguaje (se pueden usar desde distintos lenguajes de prog)
    • Pero:
      • JSON es más compacto y legible, y puede usar arrays
      • El parsing de XML es mucho más complejo
  • JSON 
    {"empleados":[
  { "nombre":"John", "apellido":"Doe" },
  { "nombre":"Anna", "apellido":"Smith" },
  { "nombre":"Peter", "apellido":"Jones" }
]}
  • XML 
    <empleados>
  <empleado>
      <nombre>John</nombre>
      <apellido>Doe</apellido>

  <empleado>
      <nombre>Anna</nombre> 
      <apellido>Smith</apellido>
  </empleado>
  <empleado>
      <nombre>Peter</nombre>

</empleados>
